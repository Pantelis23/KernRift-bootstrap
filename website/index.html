KernRift

A kernel-first systems language that turns OS invariants into compile-time errors — not boot-time crashes.

$ kernriftc --emit=elfobj --arch x86_64 uart_driver.kr -o uart_driver.o
[View on GitHub]

Generic systems languages don't model kernel reality. KernRift bakes interrupt contexts, lock ordering, MMIO semantics, capability requirements, and host-side effects directly into the type system. It bypasses LLVM entirely — emitting raw native machine code as a universal Fat Binary (.krbo), a linkable ELF object, readable assembly, or a native host executable — with zero runtime overhead and sub-millisecond compile times.

Instantaneous Compilation

KernRift skips LLVM, the C preprocessor, and every optimization IR. Compilation is 8–15x faster than GCC at object-file emission — measured over 30 runs with hyperfine.

Input Lines Tool Mean Time

Context Safety

Every function is annotated with its allowed execution contexts: boot, thread, irq, nmi, or host. Calling a yielding function from inside an interrupt handler is a compile-time failure. Calling kernel-only code from a host build tool is rejected outright. Mismatched contexts cannot link.

Effect System

Host functions declare what resources they may access via @eff(env, fs, process, stdout). The compiler verifies that effects propagate correctly across the entire call graph — a function annotated @eff(stdout) cannot silently call into code that reads environment variables without declaring it.

Global Lock Graph

Deadlocks are mathematically impossible. kernriftc link merges lock acquisition paths across all modules at link-time and rejects the binary if a lock-order cycle exists anywhere in the program — before a single byte runs.

Typed MMIO

Define hardware registers with explicit widths and rw / ro access rights. Memory-mapped I/O is volatile-safe by default, eliminating the class of optimizer bugs that silently corrupt C device drivers.

Host Execution Mode

KernRift isn't just for kernels. Ctx::Host programs compile to native host executables with --emit=hostexe, letting you write build scripts, code generators, and install tools in the same language as your drivers. build.rs replaced by build.kr.

Multiple Emit Targets

One compiler, every output format: --emit=krbo (universal Fat Binary), --emit=elfobj (linkable ELF object), --emit=asm (readable GAS assembly), --emit=hostexe (native host executable), --emit=krir (IR dump). Every target goes through the same verified pipeline.

Universal Fat Binaries

Compile once, run anywhere. kernriftc bundles x86-64 and ARM64 native machine code into a single, LZ4-compressed .krbo file with zero dynamic linker dependencies. kernrift selects and executes the right slice at runtime via mmap.

The Living Compiler

Never manually update syntax again. KernRift's migration engine maps legacy source to its canonical IR and rewrites your files to the new standard — with mathematical fitness scoring to gate code quality in CI.

Quickstart

Install 
# Linux / macOS
bash <(curl -sSf https://raw.githubusercontent.com/Pantelis23/KernRift/main/scripts/install-linux.sh)

# Windows (PowerShell) — two packages, one language
cargo install --git https://github.com/Pantelis23/KernRift kernriftc --locked
cargo install --git https://github.com/Pantelis23/KernRift kernrift --locked
uart_driver.kr — Kernel Driver 
// UART driver — only callable from interrupt or thread context.
@module_caps(Uart)

@mmio uint8 UART_DATA  = 0x3F8  rw
@mmio uint8 UART_STATUS = 0x3FD  ro

@ctx(irq, thread) @eff() @caps(Uart)
fn uart_write_byte(uint8 b) {
    mmio_write(UART_DATA, b)
}

@ctx(irq, thread) @eff() @caps(Uart)
fn uart_read_byte() -> uint8 {
    return mmio_read(UART_DATA)
}

// ✗ compile error: irq context cannot yield
// @ctx(irq) fn bad() { thread_sleep(100) }
build.kr — Host Build Script 
// Build orchestration written in KernRift itself.
// Compiled to a native host binary: kernriftc --emit=hostexe build.kr
@module_caps(Env, Process, Stdout)

@ctx(host) @eff(env)     @caps() extern fn getenv(uint64 name) -> uint64;
@ctx(host) @eff(stdout)  @caps() extern fn puts(uint64 s) -> uint32;
@ctx(host) @eff(process) @caps() extern fn system(uint64 cmd) -> uint32;
@ctx(host) @eff(process) @caps() extern fn exit(uint32 code);

@export @ctx(host) @eff(env, process, stdout)
fn main() {
    uint64 skip = getenv("SKIP_KR")
    if skip != 0 {
        puts("cargo:warning=skipping KernRift compilation")
        exit(0)
    }
    uint32 rc = system("kernriftc --emit=elfobj --arch x86_64 serial.kr -o serial.o")
    if rc != 0 { puts("cargo:error=kernriftc failed"); exit(1) }
    puts("cargo:rustc-link-arg=serial.o")
}
Compile & Run 
# Kernel driver → linkable ELF object (sub-millisecond)
$ kernriftc --emit=elfobj --arch x86_64 uart_driver.kr -o uart_driver.o

# Kernel driver → universal Fat Binary (x86-64 + ARM64)
$ kernriftc uart_driver.kr -o uart_driver.krbo
$ kernrift uart_driver.krbo

# Build script → native host executable
$ kernriftc --emit=hostexe --arch x86_64 build.kr -o build
$ ./build
cargo:rustc-link-arg=serial.o

# Dump the intermediate representation for debugging
$ kernriftc --emit=krir uart_driver.kr

# Type-check only, zero disk output
$ kernriftc check uart_driver.kr
